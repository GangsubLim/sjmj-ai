# 재학습은 수동 먼저, 실측 후 자동화한다

Phase 2의 품목 인식기 재학습은 처음엔 관리자가 수동 실행(버튼/CLI)한다. 교정이 쌓이는 동안 소규모 학습곡선을 실측해 "교정 몇 건부터 유의미한 정확도 개선이 나는가"를 확인한 뒤, 그 수치로 자동 트리거 임계를 정해 자동화한다. 금액합 자동검산 게이트도 이 자동화 단계에서 함께 붙인다.

이유: 로드맵 §5는 "교정 누적 임계 → 자동 재학습"을 그렸지만, **운영 400건 + 신규 교정 규모가 contrastive 재학습을 유의미하게 돌리기 충분한지에 대한 데이터 근거가 아직 없다**(로드맵 스스로 미검증 가정으로 명시). 임계 근거 없이 임의의 숫자로 자동 트리거를 박으면, 데이터 부족 구간에서 재학습이 오히려 정확도를 해칠 수 있다. 미검증 가정을 코드로 굳히지 않는다.

## Consequences

- 자동 트리거·검산 게이트 구현은 학습곡선 실측 결과가 나온 뒤로 미룬다. 그 전까지 재학습은 사람이 명시적으로 돌린다.
- "DB가 쌓일수록 자동 개선"은 검증 대상이지 전제가 아니다.
